JS知识总结（2）

1：JS中何时该用数组，何时该用对象？

  • a):数组： 可看做是相似数据的有序列表；如一组同事的名字or一组图书的书名；
  • b):对象：对象的最佳应用是封装单个实体的信息；如一个同事对象可能包含这个人的名字和年龄属性；一本书的对象可能有书名、作者等属性；属性是明确的命名，可以很容易的访问到，也可以给对象添加方法去操作这个对象和它的属性；
  • c):创建对象的数组：“数组性”能让脚本遍历访问整个对象列表；“对象性”能让脚本检查数组中的每个对象的特定属性；

    2: 两种从服务器获取数据到已渲染页面的方法：

  • a): Ajax方法 ：利用XMLHttpRequest对象，让已经渲染完毕的页面向服务器处理发出请求，处理返回XML文档表单中的数据，浏览器脚本能很容易的从中提取数据，更新渲染后的页面内容，这种方法被称为异步Javascript和XML,简称Ajax；
  • b):JSON方法 ：服务器处理能够将包含JS数组和自定义对象代码的字符串返给XMLHttpRequest对象，浏览器收到之后，通过一条语句(eval_r())将字符串转换成完整的JS数组和对象，以供进一步的处理，这种方法被称为JavaScript对象描述（JavaScript Object Notation）；

    10: 自定义对象的原型：

  • a):通过原型给已创建或将要创建的构造器追加属性和默认值时，所有的对象实例（包括已经创建的实例）都会自动获得这个新属性和值；
  • b):当修改某个对象实例的原型属性的值，那么这个值会成为该实例的私有属性，此操作不会影响到该对象的其他实例，其他实例依旧共享原型属性的默认值；
  • c):重载的属性值是持久的，在进行过上述修改后，就算在修改原型属性的值，这个单独分配的私有属性值是不会再收原型的变更的影响的，因为这个局部变量已经被显示重载了；
  • d) 继承原型属性的对象实例，都可以读取原型的值，即使这个值已经被对象实例重载了，constructor（或proto）属性指向维护原型信息的构造函数；
    // 自定义对象原型
    //cowoker对象构造器和他的4个实体

    function cowoker (name,age) {
    this.name = name;
    this.age = age || 0 ;
    }
    var employeeDB = new Array( );
    employeeDB[ employee.length] = new cowoker(“aaa”,23);
    employeeDB[ employee.length] = new cowoker(“bbb”,26);
    employeeDB[ employee.length] = new cowoker(“ccc”,25);
    employeeDB[ employee.length] = new cowoker(“ddd”,28);
    // 通过原型给cowoker构造器追加属性，并指定默认值
    //employeeDB数组中的每个对象实例都会立即继承这个status属性
    cowoker.prototype.status = “on duty”;
    employeeDB[2].status //on duty
    //重载某个对象实例的原型属性值；
    employeeDB[2].status = “on sick leave”;
    alert(employeeDB[1].status); //duty
    alert(employeeDB[2].status); // on sick leave
    //修改原型属性的值
    cowoker.prototype.status = “ on vacation”;
    //employeeDB[2].status因为已经被显性重载，所以不收原型的变更影响；
    alert(employeeDB[1].status); // on vacation
    alert(employeeDB[2].status); // on sick leave
    // 对象实例去读取原型的值
    alert(employeeDB[2].constructor.prototype.status); //on vacation
